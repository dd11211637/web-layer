# 协作规范

## 分支约定

- `main`：稳定主分支，用于演示和最终提交。
- `feature/chat-ui`：页面布局、输入交互、消息组件。
- `feature/api-stream`：接口调用、流式输出、引用渲染。
- `fix/*`：问题修复分支。

## 提交规范

推荐提交信息格式：

```text
feat: add chat input
feat: implement stream response reader
fix: handle empty query
docs: update task checklist
```

常用类型：

- `feat`：新增功能。
- `fix`：修复问题。
- `docs`：文档修改。
- `style`：样式调整。
- `refactor`：代码重构。
- `test`：测试相关。

## 日常开发流程

开始开发前：

```bash
git checkout main
git pull
git checkout 自己的分支
git merge main
```

提交修改：

```bash
git status
git add .
git commit -m "feat: add chat layout"
git push
```

## 合并前检查

- 页面可以正常打开。
- 输入框可以发送问题。
- Mock 或后端接口能返回回答。
- 流式输出或模拟流式输出正常。
- 引用来源能展示。
- 没有明显控制台报错。

## 冲突处理原则

- 不要直接覆盖对方代码。
- 先看冲突文件，确认双方修改目的。
- 保留双方需要的逻辑，再本地运行验证。
- 冲突较大时先沟通接口字段和组件边界。
