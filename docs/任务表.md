# Web 层两人分工与任务表

本文档用于明确两名同学如何共同维护 Web 层项目，以及 Demo 前各自需要交付什么。

状态取值建议：`待开始`、`进行中`、`已实现待验收`、`待联调`、`已完成`、`阻塞`。

## 一、分工原则

- djx 负责页面、交互、视觉呈现和用户操作体验。
- xdj 负责接口、数据结构、Mock、流式响应和异常状态。
- 共同负责引用来源展示，因为它同时涉及 UI 展示和后端字段解析。
- `main` 分支只放可运行版本；个人开发应使用功能分支，合并前先本地运行。

### 负责范围

## 六、合并与同步规则

- djx 优先在 `feature/chat-ui` 开发页面交互。
- xdj 优先在 `feature/api-stream` 开发接口和 Mock。
- 修改公共文件前先沟通，例如 `ChatPage.vue`、`src/types/chat.ts`、`docs/接口约定.md`。
- 每次合并前至少执行：

```bash
npm run build
```

- Demo 前两人共同确认：
  - 页面能打开；
  - Mock stream 能演示；
  - citations 能展示；
  - 异常状态不白屏；
  - `tests/manual-test-cases.md` 有记录。
