# Web 层接口约定

本文档约定 Web 层和 Agent 层在 Q1 阶段的聊天接口、状态码、引用字段和流式事件格式。所有 Mock 数据必须与本文档保持一致。

## 1. 接口范围

Q1 只约定一轮问答基础链路：

```text
用户输入 query -> Web 调用 Agent /api/chat -> Agent 返回 answer / citations / status / trace_id
```

Web 层不直接调用检索、向量库或 LLM 服务。

## 2. 请求格式

```http
POST /api/chat
Content-Type: application/json
```

```json
{
  "query": "项目 Q1 阶段需要完成哪些功能？",
  "session_id": "local-session-001",
  "stream": true
}
```

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `query` | string | 是 | 用户输入的问题，前端需要 trim 后校验 |
| `session_id` | string | 否 | 当前会话标识，便于后续多轮扩展 |
| `stream` | boolean | 否 | 是否请求流式输出，默认 `false` |

## 3. 普通响应格式

```json
{
  "trace_id": "trace-001",
  "status": "success",
  "answer": "Q1 阶段需要打通用户提问、检索、生成答案和展示引用的基础链路。",
  "citations": [
    {
      "citation_id": 1,
      "title": "Q1 范围说明",
      "source_url": "https://example.com/doc/1",
      "doc_id": "doc_001",
      "chunk_id": "chunk_001",
      "score": 0.86,
      "snippet": "本次迭代目标：实现一轮对话的基础功能验证..."
    }
  ]
}
```

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `trace_id` | string | 是 | 单次请求跟踪 ID，用于联调排查 |
| `status` | string | 是 | 业务状态，见状态枚举 |
| `answer` | string | 是 | 可展示给用户的答案或兜底文案 |
| `citations` | Citation[] | 是 | 引用来源数组，异常场景可为空数组 |

## 4. Citation 字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `citation_id` | number | 是 | 引用编号，从 1 开始 |
| `title` | string | 是 | 文档或片段标题 |
| `source_url` | string | 否 | 外部链接；没有链接时为空字符串或不返回 |
| `doc_id` | string | 是 | 文档 ID |
| `chunk_id` | string | 否 | 文档片段 ID |
| `score` | number | 否 | 检索相关性分数 |
| `snippet` | string | 否 | 命中文本片段 |

前端展示规则：

- `source_url` 存在时展示可点击链接；
- `source_url` 为空时不要生成空链接，展示 `doc_id` 或本地文档标题；
- `score` 只用于调试或内部展示，不作为用户主信息。

## 5. 状态枚举

| status | 含义 | 前端展示建议 |
| --- | --- | --- |
| `success` | 正常返回答案 | 展示答案和引用 |
| `invalid_query` | 用户问题为空或非法 | 提示“请输入有效问题” |
| `no_relevant_context` | 知识库没有足够信息 | 提示“当前知识库没有足够信息回答该问题” |
| `retrieval_error` | 检索服务异常 | 提示“检索服务异常，请稍后重试” |
| `llm_error` | 模型服务异常 | 提示“模型服务异常，请稍后重试” |
| `network_error` | 网络连接异常 | 提示“网络连接异常，请检查连接后重试” |
| `timeout_error` | 请求超时 | 提示“请求超时，请稍后重试” |
| `stream_error` | 流式响应中断或解析失败 | 提示“回答生成中断，请重新发送” |

## 6. 错误响应格式

Agent 层业务异常建议仍返回 HTTP 200，并用 `status` 区分；网关、鉴权或服务不可用可返回对应 HTTP 状态码。

```json
{
  "trace_id": "trace-retrieval-001",
  "status": "retrieval_error",
  "answer": "检索服务异常，请稍后重试。",
  "citations": []
}
```

前端需要兜底处理非 JSON、超时、断网和 5xx 响应，并映射为 `network_error`、`timeout_error` 或 `stream_error`。

## 7. 流式响应格式

Agent 层可使用 SSE 或 fetch stream。推荐 SSE 事件：

```text
event: meta
data: {"trace_id":"trace-stream-001","status":"success"}

event: token
data: {"text":"Q1 阶段"}

event: token
data: {"text":"需要打通用户提问、检索、生成答案和展示引用。"}

event: citations
data: [{"citation_id":1,"title":"Q1 范围说明","source_url":"https://example.com/doc/1","doc_id":"doc_001","chunk_id":"chunk_001","score":0.86}]

event: done
data: {"trace_id":"trace-stream-001","status":"success"}
```

事件类型：

| event | data | 说明 |
| --- | --- | --- |
| `meta` | `{ trace_id, status }` | 开始阶段返回请求元信息 |
| `token` | `{ text }` | 增量答案文本 |
| `citations` | `Citation[]` | 引用列表 |
| `error` | `{ trace_id, status, message }` | 流式过程中发生错误 |
| `done` | `{ trace_id, status }` | 流式结束 |

## 8. Web 层处理要求

- 发送前 trim `query`，空字符串不请求 Agent；
- 每次请求生成或传入 `session_id`；
- 所有响应都要记录或展示 `trace_id`；
- `citations` 为空时隐藏引用列表或展示无引用状态；
- 任何错误状态都不能导致页面白屏或 loading 死锁；
- Mock 数据变更后必须同步更新 `docs/Mock数据说明.md`。
