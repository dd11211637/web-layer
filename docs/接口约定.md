# Web 层接口约定

## 聊天请求

```http
POST /api/chat
Content-Type: application/json
```

请求体：

```json
{
  "query": "项目 Q1 阶段需要完成哪些功能？"
}
```

## 普通响应

```json
{
  "trace_id": "trace-001",
  "answer": "Q1 阶段需要打通用户提问到展示引用的完整链路 [1]。",
  "citations": [
    {
      "citation_id": 1,
      "title": "Q1 范围说明",
      "source_url": "",
      "doc_id": "q1_scope",
      "chunk_id": "chunk_001",
      "score": 0.86
    }
  ],
  "status": "success"
}
```

## 状态码约定

| status | 含义 | 前端展示 |
| --- | --- | --- |
| `success` | 成功回答 | 展示答案和引用 |
| `invalid_query` | 问题为空或非法 | 提示用户输入问题 |
| `no_relevant_context` | 知识库信息不足 | 展示知识库没有足够信息 |
| `retrieval_error` | 检索服务异常 | 展示检索服务暂时不可用 |
| `llm_error` | 模型服务异常 | 展示模型服务暂时不可用 |

## 流式响应建议

后端可以使用 SSE 或 fetch stream。

推荐事件：

```text
event: token
data: {"text":"Q1 阶段"}

event: citations
data: [{"citation_id":1,"title":"Q1 范围说明","source_url":"","doc_id":"q1_scope","chunk_id":"chunk_001","score":0.86}]

event: done
data: {"trace_id":"trace-001","status":"success"}
```
