# Web 层 Demo 验收清单

本文档用于 Q1 Demo 前确认 Web 层是否达到“可展示、可联调、可兜底”的要求。

## 一、Demo 目标

Web 层 Demo 需要展示用户层已经具备以下能力：

```text
用户输入问题
-> 前端发送请求
-> 页面展示回答生成过程
-> 展示最终回答
-> 展示引用来源
-> 异常状态可读兜底
```

本次 Demo 不验收复杂账号体系、多轮会话历史、权限过滤、文档处理、检索算法和 LLM Prompt 质量。

## 二、Demo 演示路径

### 路径 1：正常问答

操作：

1. 打开 Web 页面。
2. 输入问题：`项目 Q1 阶段需要完成哪些功能？`
3. 点击发送或按 Enter。

验收标准：

- 页面显示用户问题。
- 页面出现生成中状态。
- 回答逐步显示，或至少完整展示。
- 回答正文中出现引用编号，例如 `[1]`。
- 引用来源区域展示标题、doc_id、chunk_id 或 source_url。
- 响应结束后 loading 消失。
- 页面展示或记录 `trace_id`。

### 路径 2：空输入拦截

操作：

1. 输入空字符串或纯空格。
2. 点击发送或按 Enter。

验收标准：

- 前端不发送请求。
- 页面提示请输入有效问题。
- 页面不新增空白消息。

### 路径 3：无相关文档

操作：

1. 输入包含“无相关”的测试问题，或切换到 no relevant context Mock。

验收标准：

- 页面展示“当前知识库没有足够信息回答该问题”。
- citations 为空时页面不报错。
- 用户可以继续发送下一条问题。

### 路径 4：服务异常兜底

操作：

1. 输入包含“检索异常”的测试问题，触发 retrieval error。
2. 输入包含“模型异常”的测试问题，触发 llm error。

验收标准：

- 检索异常时展示检索服务异常提示。
- 模型异常时展示模型服务异常提示。
- 页面不白屏、不死锁。
- 输入框和发送按钮恢复可用。

### 路径 5：引用无链接

操作：

1. 输入包含“无链接”的测试问题，触发 citation without url Mock。

验收标准：

- 引用来源仍展示标题。
- source_url 为空时不生成空链接。
- 页面展示 doc_id 和 chunk_id，保证来源可追溯。

## 五、Demo 前检查顺序

1. djx 检查页面是否能打开、布局是否正常、输入交互是否正常。
2. xdj 检查 Mock 数据、接口字段、流式输出、引用展示和异常状态。
3. 两人一起跑一遍“正常问答 -> 异常兜底 -> 清空对话”的完整演示路径。
4. 在 `tests/manual-test-cases.md` 写下测试日期、测试人、提交号、通过项和失败项。
5. 若真实 Agent 不稳定，Demo 使用 Mock Client 兜底，但需要明确说明接口字段已按 Agent 约定设计。
