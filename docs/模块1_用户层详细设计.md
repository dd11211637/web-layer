# 模块 1：用户层详细设计

## 1. 背景

AI 知识问答系统需要一个面向用户的 Web 入口，用于承接提问、展示 Agent 层返回的答案和引用来源。Q1 阶段重点不是完整产品化，而是打通可演示、可联调、可验收的一轮问答链路。

## 2. 模块目标

用户层需要完成：

- 提供 Web 聊天页面；
- 支持用户输入问题并发送；
- 调用 Agent 层 `/api/chat` 接口；
- 展示普通响应或流式响应；
- 展示引用来源；
- 处理无相关文档、检索异常、模型异常、网络异常等状态；
- 暴露 `trace_id`，便于联调定位；
- 提供 Mock 数据，保证前端可以独立开发和 Demo。

## 3. 非目标

Q1 用户层不负责：

- 文档解析和清洗；
- 向量化和入库；
- 检索召回策略；
- Prompt 组装；
- LLM 调用；
- Agent 工具编排；
- 用户登录、权限和审计。

## 4. 核心链路

```text
用户输入问题
  -> 前端 trim 和空输入校验
  -> Web 调用 Agent /api/chat
  -> Agent 返回 answer / citations / status / trace_id
  -> Web 展示答案、引用和 trace_id
  -> 异常状态展示兜底提示
```

## 5. 页面组成

| 组件 | 职责 |
| --- | --- |
| `ChatPage.vue` | 聊天页面容器，管理消息、loading、错误和清空逻辑 |
| `ChatInput.vue` | 输入框、发送按钮、Enter 和 Shift + Enter 交互 |
| `MessageList.vue` | 消息列表渲染 |
| `MessageBubble.vue` | 单条用户或助手消息展示 |
| `CitationList.vue` | 引用列表容器 |
| `CitationCard.vue` | 单条引用展示，处理链接和无链接场景 |
| `ErrorMessage.vue` | 错误状态提示 |
| `LoadingIndicator.vue` | 请求中或流式输出状态 |

## 6. 数据模型

核心类型放在 `src/types/chat.ts`：

- `ChatRequest`：请求体；
- `ChatResponse`：普通响应；
- `ChatStatus`：业务状态枚举；
- `Citation`：引用来源；
- `ChatMessage`：页面消息结构；
- `ChatStreamEvent`：流式事件结构。

## 7. 接口设计

普通请求：

```json
{
  "query": "项目 Q1 阶段需要完成哪些功能？",
  "session_id": "local-session-001",
  "stream": false
}
```

普通响应：

```json
{
  "trace_id": "trace-001",
  "status": "success",
  "answer": "Q1 阶段需要打通用户提问、检索、生成答案和展示引用的基础链路。",
  "citations": []
}
```

流式输出建议使用 SSE 事件：`meta`、`token`、`citations`、`error`、`done`。

## 8. 状态处理

| 状态 | 页面行为 |
| --- | --- |
| `success` | 展示答案和引用 |
| `invalid_query` | 阻止发送或展示输入错误 |
| `no_relevant_context` | 展示知识库信息不足提示 |
| `retrieval_error` | 展示检索服务异常提示 |
| `llm_error` | 展示模型服务异常提示 |
| `network_error` | 展示网络连接异常提示 |
| `timeout_error` | 展示请求超时提示 |
| `stream_error` | 展示流式响应中断提示 |

## 9. Mock 设计

Mock 数据放在仓库根目录 `mock/`，覆盖：

- 正常完整回答；
- 流式响应事件；
- 知识库无相关内容；
- 检索异常；
- 模型异常；
- 空输入或非法问题；
- 本地文档引用没有 `source_url`。

前端开发时可以使用 `src/mock/mockChatClient.ts` 读取 Mock 数据，暂时不依赖真实 Agent。

## 10. 验收方式

Q1 验收至少覆盖：

- 页面加载；
- 正常问答；
- 空输入；
- Enter 和 Shift + Enter；
- 流式输出；
- 引用展示；
- 无相关文档；
- 检索异常；
- 模型异常；
- 网络异常；
- 清空对话；
- 复制回答；
- `trace_id` 记录。

测试记录放在 `tests/manual-test-cases.md` 和 `tests/integration-test-record.md`。
